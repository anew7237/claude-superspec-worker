# B-narrow Toolchain Cleanup — Design Doc

**Date**: 2026-04-30
**Status**: Approved (brainstorming) — pending implementation plan
**Predecessor**: `.docs/20260430a-cloudflare-worker.md` (the Cloudflare Worker variant working doc; informs the broader R1 plan)

## Purpose

Prepare a clean baseline for the **R1 path** — full re-design via spec-kit pipeline rather than importing nodejs sibling's `001-superspec-baseline` artefacts. R1 will subsequently:

```
/speckit.constitution          → write a constitution that natively reflects
                                 Node + Worker monorepo (no Variant Amendment;
                                 the two-runtime reality is built into the
                                 main constitution body)
/speckit.specify 001-superspec-baseline
                               → regulate the current Node baseline + monorepo
                                 structure as a fresh spec-kit feature
/speckit.specify 002-cloudflare-worker
                               → drive Worker variant from .docs/20260430a-cloudflare-worker.md
```

This design covers the **pre-spec-kit** preparatory work only — three small commits that put the repo in the right state for `/speckit.constitution` to start clean.

## Scope

**In:** identity rename, gitignore/prettierignore gaps, vitest 2→4 bump, constitution revert to spec-kit template.

**Out (deferred to R1 spec-kit pipeline):**
- Constitution content
- Any spec deliverable
- Variant Amendment (R1 will fold its substance into the main constitution body, so no amendment will exist)
- Any Worker-side artefact (`src/worker/`, `wrangler.jsonc`, `.dev.vars`, worker tsconfig, worker vitest config)
- Toolchain split (dual tsconfig / dual vitest config) — these belong to 002 per `.docs/20260430a-cloudflare-worker.md` §5.8

### Commit 2 — Vitest 2 → 4 bump

Migrates the test runner across a major-version bump while no Worker code yet exists, so blast radius is limited to the 8 existing Node-side test files.

| File | Change |
|---|---|
| `package.json` | `"vitest": "^2.1.0"` → `"vitest": "^4.0.0"` |
| `pnpm-lock.yaml` | Regenerated by `pnpm install` |
| `tests/node/http-metrics.bench.ts` | Adjust if vitest 4's bench API requires it (expected ≤ 5 lines based on API surface scan; sit-rep to user if larger) |

**API-surface scan (already done; recorded for traceability):**

| Vitest API | Used in | v2→v4 risk |
|---|---|---|
| `describe / it / expect / beforeEach / afterEach` | all 8 files | none |
| `vi.resetModules()` / `vi.stubEnv()` / `vi.unstubAllEnvs()` / `vi.spyOn()` | `http-metrics.opt-out.test.ts`, `http-metrics.test.ts` | none (stable since vitest 0.34+) |
| `bench()` from `vitest` | `http-metrics.bench.ts` | low–medium (tinybench v2→v3 under the hood; reporter / threshold config schema may shift) |

**Verification commands (in order):**
1. `pnpm install` — succeeds with new lockfile
2. `pnpm test` — 20 / 20 pass (matches pre-bump baseline)
3. `pnpm typecheck` — `tsc --noEmit` exit 0
4. `pnpm bench --run` — completes without errors; SC-007 threshold check still meaningful

If `pnpm bench --run` fails or yields a config schema error that needs more than ~5 lines of fix in `http-metrics.bench.ts`, halt and confirm with user before continuing.

## Risks and mitigations

1. **Vitest 4 bench API breaks `http-metrics.bench.ts`** — Probability: medium. Mitigation: small in-place fix (≤5 lines expected). Halt-condition: if fix needs > 5 lines or new config file, stop and confirm scope with user before continuing.
2. **Constitution revert deletes 339 lines of content** — Probability: 100% (intentional). Mitigation: content remains in commit `b9fba00` for retrieval; R1 will reference both the deleted content and `.docs/20260430a-cloudflare-worker.md` §5.7 when drafting the new constitution.
3. **`pnpm install` resolves vitest 4 with peer-dep conflicts** — Probability: low. Mitigation: `--frozen-lockfile=false` for the install (default after manual `package.json` edit); inspect resolution warnings; if a peer dep (e.g., `@vitest/expect`, `@vitest/coverage-v8`) needs explicit version, add to `devDependencies` in same commit.

## Out of scope (explicit)

- Anything to do with Worker runtime
- Tsconfig structure
- Vitest config file(s)
- `package.json` script shape (no `dev:node` / `dev:worker` split)
- Worker devDependencies (`wrangler`, `@cloudflare/workers-types`, `@cloudflare/vitest-pool-workers`)
- Constitution content
- Any spec deliverable
- 001 import from nodejs sibling
