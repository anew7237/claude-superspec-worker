# B-narrow Toolchain Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Put the repo in a clean state for the R1 spec-kit pipeline by renaming the package, closing two ignore-list gaps, bumping vitest 2 → 4, and reverting the constitution to its spec-kit template.

**Architecture:** Three independent commits in order — text-only edits first (zero risk), the toolchain bump second (largest blast radius, isolated commit with full quality-gate verification), and the constitution revert last (single-file replace with `git show`). No new files created beyond this plan; no Worker-side artefacts touched.

**Tech Stack:** pnpm 9, vitest 2 → 4, Node 22, prettier 3, TypeScript 5.7. Test runner upgrade is the only behaviour-affecting change.

**Spec:** `.docs/20260430b-b-narrow-toolchain-cleanup-design.md` (commit `b971cfe`).
